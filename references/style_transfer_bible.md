# 源片风格圣经交付规则

## 目的

源片风格圣经是给其他 AI 使用的风格知识库，不是普通拉片报告的摘要。它的任务是让另一个 AI 读完后，能迅速掌握源片的画面风格，并据此生成类似风格的场景图、视频提示词，或在后续任务中迁移到新剧本。

默认目标：
- 第一优先级：生成相似风格的场景图。
- 第二优先级：生成可用的视频提示词。
- 第三优先级：迁移到新剧本时不硬套原片。

## 默认交付形态

三交付的触发边界以 `SKILL.md` 中 "default three-deliverable package" 一节为准：快速看一下、局部段落、纯截图/剧本任务默认不触发；用户单独只要风格圣经时，先准备抽帧证据层，基础报告全文可省略但必须注明证据来源。本文件只负责风格圣经本身的写法。

完整本地视频拉片默认三交付：
1. 帧/自适应详细拉片报告：保留时间码、证据帧、画面变化、字幕/声音线索，是所有后续分析的基础。
2. 专业导演分析：解释为什么这样拍、导演意图、观众心理、视听执行和可学习方法。
3. 源片风格圣经：解释风格 DNA、视觉资产、生成提示词和可复用规则，专门服务风格迁移。

源片风格圣经必须独立成文，不要塞在主拉片报告或专业导演分析后半部分。默认文件名：

```text
片名_源片风格圣经_v01.md
片名_源片风格圣经_嵌图版_v01.docx
```

Word 版本必须嵌入图片。Markdown 可以用相对路径引用图片，但不能把 Markdown 路径版当作最终给其他 AI 上传的稳定交付。

长度目标：中版，约 20-50 页。控制图片数量，不要把逐秒抽帧全部嵌入 Word。

## 素材选择原则

图片按影片段落分类，而不是按元素冷分类。每个段落选 6-10 张关键图，覆盖该段的风格推进：
- 段落起点图。
- 主体第一次清晰出现。
- 空间/权力/威胁关系最明确的图。
- 光影、色彩、材质最能代表该段的图。
- 情绪转折图。
- 段落收束图。

短视频可以少于 6 张；长视频不要机械每段塞满 10 张，优先选风格密度高、可迁移价值高的帧。

选图间隔按内容自适应，不按固定逐秒硬塞：
- 常规段落可约 2 秒选一张代表图。
- 字幕、表情、动作、构图或光影快速变化时，可回到 1 秒或 0.5 秒关键图。
- 静态、重复、长时间同构图的段落只选首/中/尾代表图。
- 风格圣经不是逐秒拉片证据库；它只嵌入对学习风格有价值的展示帧和资产图。

图片在风格圣经里不是单纯证据，而是视觉资产索引。每张图旁边要说明它可以被其他 AI 学到什么。

## 核心文档结构

```markdown
# 《片名》源片风格圣经
生成者：啪趴虾
用途：供其他 AI 学习源片视觉风格、生成同风格场景图和视频提示词。

## 1. 风格定位总述
## 2. 风格 DNA 总表
## 3. 影片段落式图片资产索引
## 4. 场景级风格资产卡
## 5. 元素级风格资产卡
## 6. 图像生成变量模板
## 7. 视频生成变量模板
## 8. 风格保持与跑偏警报
## 9. 最终总结
```

不要默认加入“风格复刻测试题”。用户明确要求评测另一个 AI 是否读懂时，再添加测试题。

## 1. 风格定位总述

用 3-6 段说明源片最核心的风格气质。重点回答：
- 这条片子最容易被识别的画面风格是什么？
- 它靠哪些画面规律建立这种风格？
- 它的风格服务什么情绪和叙事功能？
- 哪些元素只是表层装饰，不能被误认为风格核心？

写法要偏教材式，而不是短摘要。给其他 AI 足够多的因果解释。

## 2. 风格 DNA 总表

建议字段：

| 维度 | 风格规则 | 叙事/情绪功能 | 可迁移方向 |
|---|---|---|---|
| 构图 |  |  |  |
| 光影 |  |  |  |
| 色彩 |  |  |  |
| 材质 |  |  |  |
| 空间 |  |  |  |
| 主体轮廓 |  |  |  |
| 天气/环境颗粒 |  |  |  |
| 镜头距离 |  |  |  |
| 运动感 |  |  |  |

剧情和风格必须强绑定。不要只写“冷色、逆光、低角度”，要说明它们分别承担压迫、孤独、威慑、神圣、逃亡、决战、回望等功能。

## 3. 影片段落式图片资产索引

每个段落先写一段解释，再列图片资产表。

```markdown
### 00:00-00:20 段落名

**剧情功能：** 这段发生了什么，人物处在什么状态。

**风格功能：** 这段的画面为什么这样设计，它让观众感到什么。

| 时间 | 参考图 | 画面风格资产 | 可学习点 | 可迁移用途 |
|---:|---|---|---|---|
| 00:05 | ![00:05](<../01_逐秒抽帧/frames/frame_000006.jpg>) |  |  |  |
```

每段 6-10 张图。避免逐秒表。风格圣经要帮助 AI 学风格，不是复查每一秒；展示帧可以在 0.5-2 秒之间变化，由内容密度决定。

## 4. 场景级风格资产卡

场景级资产卡用于复用整体画面氛围，例如“雪中宫门压迫场景”“金色大殿放逐场景”“巨兽低空压城场景”“废墟夕阳告别场景”。

每张卡建议结构：

```markdown
### 场景资产卡 S01：资产名

**来源段落 / 参考帧：** 00:00-00:20，frame_000001、frame_000006、frame_000012

**核心视觉 DNA：** 这一场景最关键的构图、光影、色彩、空间、材质规律。

**叙事/情绪功能：** 这个场景不是为了好看，而是在故事里承担什么功能。

**构图规则：** 主体位置、前中后景、遮挡、尺度关系、视线引导。

**光影色彩规则：** 主光源、冷暖关系、阴影比例、雾/雪/火/尘如何参与画面。

**材质与细节：** 石墙、金属、鳞片、布料、尘土、雪、火、湿冷空气等。

**可迁移元素：** 可迁移到别的剧本的画面规律。

**易跑偏点：** AI 容易只学到哪些表皮，或把风格用错到什么场景。

**图像生成变量模板：**
`主体={角色/怪物/建筑}，场景={空间}，情绪={压迫/孤独/神圣/逃亡/决战}，构图={远景小人物/低角度仰拍/前景遮挡/门框压迫}，光影={冷灰天光/火光逆光/雾中轮廓光}，材质={石墙/鳞片/铁锈/尘土/雪/湿冷布料}，风格={源片风格关键词}`

**视频生成变量模板：**
`镜头从{起点构图}开始，{主体}在{场景}中{动作}，镜头以{运动方式}强化{情绪功能}，光影保持{光影规则}，环境颗粒为{雪/雾/烟/尘/火星}，结尾落在{情绪收束画面}。`
```

Prompt 默认中文为主。不要默认写英文版本，除非用户要求或指定更适合英文的模型。

## 5. 元素级风格资产卡

元素级资产卡采用分优先级覆盖。

一级核心元素：必须拆。它决定风格识别度，例如主角剪影、巨型怪物/龙、权力建筑、宫门、王座、战场、废墟、神圣光束。

二级辅助元素：按需拆。它增强质感和迁移稳定性，例如雪、烟、火、灰尘、鳞片、铁锈、湿冷布料、残墙、旗帜、手持道具。

三级气氛元素：合并讲。不要每个都单独成卡，例如雾、远山、低云、背景人群、环境颗粒，可写入场景资产卡。

元素级资产卡建议结构：

```markdown
### 元素资产卡 E01：元素名

**优先级：** 一级核心 / 二级辅助 / 三级气氛合并项

**视觉识别点：** 轮廓、比例、材质、颜色、运动方式。

**叙事/情绪功能：** 它在源片里代表什么，不要只写外观。

**可迁移方向：** 适合迁移到哪些新角色、新怪物、新建筑、新道具或新场景。

**变量模板：**
`元素={新元素}，轮廓={轮廓规则}，材质={材质规则}，尺度={与人物/环境的比例}，情绪功能={威慑/守护/审判/诱惑/灾难/记忆}，光影={光影规则}`

**跑偏警报：** 哪些替换会破坏源片风格。
```

## 6. 图像生成变量模板

文末集中给可复用变量模板，而不是只给一次性 prompt。默认中文。

基础模板：

```text
主体={新剧本角色/怪物/建筑/道具}
场景={宫门/荒野/废墟/山谷/战场/展厅/密林/王座厅}
情绪={压迫/孤独/神圣/逃亡/决战/告别/回望}
构图={远景小人物/低角度仰拍/前景遮挡/门框压迫/巨物压顶/背影面对大景}
光影={冷灰天光/金色侧逆光/火光逆光/雾中轮廓光/低照度顶光}
色彩={主色/辅助色/冷暖对比}
材质={石墙/鳞片/铁锈/尘土/雪/湿冷布料/烟尘/残破金属}
空间关系={人物与权力建筑/巨物/自然环境的尺度关系}
画面目的={让观众感到什么，并推动什么剧情功能}
```

## 7. 视频生成变量模板

视频模板必须包含镜头、动作、节奏、情绪推进，而不是只写静态画面。

```text
镜头起点={开场构图}
主体={角色/怪物/建筑}
动作={走出/逼近/停顿/低头/展开翅膀/转身/倒下/被搀扶}
镜头运动={缓慢推进/低角度仰拍/横向跟随/由近到远拉开/从遮挡后露出}
节奏={压抑慢节奏/突然加速/静默停顿/高潮释放}
环境变化={雪加密/火光增强/雾散开/尘土升起/远处光线打开}
情绪弧线={起点情绪 -> 转折情绪 -> 结尾情绪}
结尾画面={可作为海报或关键帧的收束图}
```

## 8. 风格保持与跑偏警报

必须写一个短章节列出：
- AI 最容易误解的表面元素。
- 哪些场景不适合套用该风格。
- 哪些变量可以替换，哪些变量替换后会破坏风格。
- 如果源片是知名 IP，默认不强制写版权避险；只有用户要求更安全迁移时，再加入“可迁移 / 不可照搬 / 相似度避险”表。

## 交付与 QA

`qa_lapian_delivery.py` 在 `--project-dir` 模式下会自动审计风格圣经：按文件名（含"风格圣经"）识别后，检查必备章节（风格定位、DNA、资产索引、场景级、元素级、图像生成、视频生成、跑偏）、资产卡数量和图片引用；缺章节或零图片会判为失败。"DOCX 嵌图数量与选用图片数量基本一致"这一条仍需人工抽查，脚本只验证嵌图数量大于 0。

风格圣经交付前至少检查：
- Markdown 存在且 UTF-8 可读。
- Word DOCX 存在且图片真实嵌入。
- DOCX 内 `word/media/` 图片数量大于 0，并与风格圣经中选用图片数量基本一致。
- DOCX 内存在 `a:blip` 图片引用。
- 段落图片索引存在，且每个主要段落通常有 6-10 张关键图。
- 图片选择说明中明确它不是逐秒全量嵌图，而是 0.5-2 秒自适应关键图。
- 文档包含场景级资产卡、元素级资产卡、图像变量模板、视频变量模板。
- 如果未能生成 DOCX，最终回复必须说明缺口，不能把 Markdown 路径版说成已完成的 AI 知识库 Word。
